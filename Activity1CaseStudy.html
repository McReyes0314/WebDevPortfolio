Process

Throughout this group activity, we are assigned to explore a case study and provide solutions by answering specific questions for both the front-end and back-end teams. Here are the details for our activity:

  • Problem: The front-end team encounters issues with retrieving blog posts from the Laravel API, resulting in unexpected responses and difficulties displaying the content.
  • Questions for Front-End:
    1. Identify three specific issues you might face with the API responses.
    2. Propose two debugging strategies you would use to resolve these issues.
    3. Describe how you would communicate these issues to the Back-End Team, including the information you would provide.
  • Questions for Back-End:
    1. Outline three best practices you should follow when designing the API endpoints in Laravel.
    2. Explain how you would implement error handling in your API to assist the Front-End Team in troubleshooting.
    3. Describe one testing methodology you would use to ensure that your API is functioning correctly before the Front-End Team integrates it.

Learnings

Throughout the group activity, I learned the following aspects, especially in back-end development:

  • The best practices to follow when designing APIs, such as using "Resource-Based Routing" to ensure efficient client-server interaction through requests and responses.
  • The importance of using built-in validation features for input validity and output formatting. This improves data consistency and reduces security vulnerabilities.
  • The significance of implementing robust error handling by providing detailed error messages and maintaining a consistent error response structure.
